# 数据处理 Prompt(给 Claude Code 用)

把这段话丢给 Claude Code,让它帮你把《小红书古风提示词合集.md》转成完整 JSON。

---

## 给 Claude Code 的 Prompt

```
晶晶,我要把这个 md 文件转成结构化 JSON,用在我的古风提示词生成器网站上。

文件路径:小红书古风提示词合集.md

转换规则:
1. 每个一级标题(# 开头)是一个主题 theme
2. 每个二级标题(## 开头,数字编号)是该主题下的一个变体 variant
3. 二级标题下面的那段文字就是完整 prompt

需要你做的加工:
1. 给每个主题加一个英文 id(拼音缩写即可,比如"赤梦流霞" → "chi_meng_liu_xia")
2. 给每个主题提炼一个 20 字以内的 tagline(从它第一个变体的氛围描述里抽)
3. 给每个主题选一个主色 HEX(根据主题的色彩关键词判断:赤红/淡紫/金/蓝/绿/粉/墨黑等)
4. 给每个变体提炼一个 4-6 字的 poseLabel(姿势标签),从 prompt 中找关键动作:
   - "正面站立,双手交叠" → "正面伫立"
   - "回头" → "回眸瞬间"
   - "旋身" → "旋身动态"
   - "特写" → "面部特写"
   - "手持剑" → "持剑而立"
   - "坐于石上" → "静坐一瞬"
   - 以此类推,要贴合实际动作
5. 给每个变体提炼一个 20 字以内的 description(只描述这个变体独特的动作/构图差异)
6. prompt 字段保留原文完整内容,不要删改

输出格式:
{
  "themes": [
    {
      "id": "...",
      "name": "...",
      "tagline": "...",
      "color": "#......",
      "variants": [
        {
          "id": "v1",
          "poseLabel": "...",
          "description": "...",
          "prompt": "......"
        }
      ]
    }
  ]
}

输出文件:themes.json(保存到 outputs 目录,方便我下载)

完成后告诉我:
- 总共处理了多少个主题
- 总共多少个变体
- 有没有哪些变体的 prompt 内容过长或需要我手动复核的
```

---

## 使用建议

1. 先把 md 和这段 Prompt 都发给 Claude Code
2. 让它先处理 3 个主题看效果,确认 poseLabel 提炼质量
3. 如果满意,再让它跑完全部 42 个
4. 生成的 themes.json 直接替换到网页项目里
